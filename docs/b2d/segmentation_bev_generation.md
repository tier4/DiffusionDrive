# Segmentation BEV Generation for Bench2Drive

This guide describes the segmentation-based BEV map generation system for the Bench2Drive dataset, which provides significantly improved road coverage compared to vector-based generation.

## Key Benefits

- **5.6x better coverage**: ~20,500 pixels per frame vs ~3,600 for vector mode
- **Filled road surfaces**: Complete drivable area representation
- **Improved training signal**: Dense semantic maps for better model learning

## Usage

### Generating Segmentation BEV Cache

```bash
# Basic usage (recommended)
python3 scripts/generate_bev_cache.py \
    --data-root /workspace/Bench2Drive \
    --map-dir /workspace/Bench2Drive-Map \
    --output-dir "${NAVSIM_EXP_ROOT}/bench2drive_bev_cache_seg" \
    --generation-type segmentation

# With optimized parameters
python3 scripts/generate_bev_cache.py \
    --data-root /workspace/Bench2Drive \
    --map-dir /workspace/Bench2Drive-Map \
    --output-dir "${NAVSIM_EXP_ROOT}/bench2drive_bev_cache_seg" \
    --generation-type segmentation \
    --lane-width 5.0 \
    --fill-drivable \
    --workers 16
```

### Key Parameters

- `--generation-type segmentation`: Enables segmentation mode
- `--lane-width 5.0`: Lane width in meters (default: 3.5, recommended: 5.0)
- `--fill-drivable`: Fill entire drivable areas (enabled by default)
- `--workers`: Number of parallel workers for faster processing

## Technical Details

### BEV Classes Generated

1. **Background** (0): Non-drivable areas
2. **Road** (1): All drivable surfaces including lanes
3. **Walkway** (2): ⚠️ Not generated (see limitations)
4. **Centerline** (3): Center lane markings
5. **Static** (4): Traffic lights, stop signs
6. **Vehicle** (5): Not generated by map
7. **Pedestrian** (6): Not generated by map

### Implementation Approach

The segmentation generator:
1. Draws all lanes with 5m width to ensure overlap
2. Applies morphological closing to fill gaps
3. Draws centerlines and static objects on top
4. Generates consistent 256×256 full BEV maps

### Known Limitations

**Bench2Drive HD maps lack semantic lane types:**
- Cannot distinguish sidewalks from roads
- No parking area information
- No bike lane markings

This is a dataset limitation, not an implementation issue. The maps only contain:
- Lane IDs (numeric positions)
- Marking types (Solid, Broken, Center)
- Trigger volumes (traffic lights, stop signs)

## Comparison with Vector Mode

| Aspect | Vector Mode | Segmentation Mode |
|--------|------------|-------------------|
| Coverage | ~3,600 pixels/frame | ~20,500 pixels/frame |
| Representation | Lane lines only | Filled road surfaces |
| File Size | Smaller | Larger (but compressed) |
| Training Performance | Baseline | Improved (denser signal) |
| Generation Speed | Faster | Slightly slower |

## File Structure

- `navsim/common/bev_generation_factory.py`: Factory for creating generators

### Related Scripts
- `scripts/generate_bev_cache.py`: Cache generation script
- `scripts/test_segmentation_15samples.py`: Visual comparison tool

## Troubleshooting

### If roads appear disconnected:
- Increase `--lane-width` parameter (try 6.0 or 7.0)
- Check that `--fill-drivable` is enabled

### If generation is slow:
- Increase `--workers` parameter
- Use Ray distributed processing for large datasets

### To verify generation quality:
```bash
python3 scripts/test_segmentation_15samples.py
```
This creates a visual comparison of vector vs segmentation modes.
